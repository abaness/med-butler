## 二、项目背景与立项依据

### 2.1 业务背景

中国已进入"慢病时代"。根据《中国慢性病防治中长期规划（2017-2025 年）》及相关公开数据：

- 全国慢性病患者超 **4 亿人**，高血压、糖尿病、高血脂"三高"人群占比最高；
- 慢病导致的死亡占我国总死亡人数的 **88%**，导致的疾病负担占疾病总负担的 **70% 以上**；
- **60% 以上**的慢病患者存在"漏服药、错服药、擅自停药"的情况，直接影响治疗效果；
- 一线城市慢病患者线上问诊、线上购药的渗透率已超 **35%**，且持续快速增长。

### 2.2 业务痛点

通过对美团买药、美团问诊现有业务链路分析，发现"问诊→购药→用药→续方"四个环节相互割裂：

| 环节 | 现状 | 痛点 |
| :--- | :--- | :--- |
| 问诊 | 图文/视频问诊独立进行 | 医嘱未形成结构化数据，无法指导后续用药 |
| 购药 | 按订单维度下单 | 同一药品多次重复下单，平台难以识别慢病用户 |
| 用药 | 用户自行管理 | 漏服/错服率高，无提醒闭环 |
| 续方 | 用完再临时问诊 | 断药/延误治疗，体验差 |

### 2.3 立项价值

| 维度 | 价值点 |
| :--- | :--- |
| **用户价值** | 降低漏服率，提升慢病控制达标率，建立健康依赖感 |
| **商业价值** | 提升药品复购率、LTV；锁定高价值慢病人群；带动问诊、健康商城、保险等变现 |
| **平台价值** | 填补美团医药在"用药管理"环节的产品空白，形成闭环壁垒 |
| **社会价值** | 响应"健康中国 2030"战略，助力分级诊疗与慢病管理 |

## 三、市场与用户研究

### 3.1 市场规模

- 国内慢病管理市场规模已突破 **千亿级别**，年复合增长率超 **15%**；
- 其中"数字化慢病管理"是增速最快的细分赛道；
- 政策端持续利好：互联网医院、电子处方、医保线上支付等基础设施成熟。

### 3.2 竞品速览

| 竞品 | 核心能力 | 不足 |
| :--- | :--- | :--- |
| 叮当快药"健康管家" | 用药提醒、快药配送 | 问诊薄弱，AI 能力弱 |
| 京东健康"家庭医生" | 专病管理、名医问诊 | 履约时效弱于美团 |
| 好大夫在线 | 医生资源强 | 无购药/配送能力 |
| 微信"药顾问" | 提醒轻便 | 无购药闭环 |

**美团差异化优势**：**30 分钟极速达 + 本地生活流量 + 骑手网络 + 问诊资源**，在"用完即送"的续方场景具备天然壁垒。

### 3.3 用户调研洞察（定性访谈 N=12，定量问卷 N=200）

TOP 5 痛点（按提及率排序）：

1. **"忘记吃药"**（78%）——尤其是中老年人、上班族晚餐后那顿药
2. **"不知道什么时候该去复诊/续方"**（65%）
3. **"家里药箱一团乱，不知道还剩多少"**（54%）
4. **"多种药同时吃，怕冲突"**（41%）
5. **"家人（父母）吃药情况我也想知道"**（38%）

## 四、产品定位与目标

### 4.1 一句话定位

> **"美团慢病用药小管家——让慢病用药，不再靠记性。"**

一个基于 AI 的慢病用药全周期管理工具，在美团医药体系内，将"问诊—购药—用药—续方"四环打通，让用户只需专注按时吃药，其他交给管家。

### 4.2 产品目标（SMART）

| 目标类型 | 指标 | 上线 3 个月 | 上线 6 个月 |
| :--- | :--- | :--- | :--- |
| **用户目标** | 慢病用药计划创建用户数 | 50 万 | 200 万 |
| **行为目标** | 用药提醒签到率 | ≥ 65% | ≥ 75% |
| **业务目标** | 慢病药品复购率 | +15% | +30% |
| **业务目标** | 续方转化率（提醒→下单） | ≥ 20% | ≥ 35% |
| **满意度** | NPS | ≥ 40 | ≥ 55 |

### 4.3 设计原则

1. **极简输入**：OCR/AI 识别代替手动录入，让老人也能用。
2. **主动服务**：管家"推"给你，而不是你"找"它。
3. **闭环优先**：每一次提醒都能一键完成相关动作（复购、复诊、咨询）。
4. **有温度**：语气不要像 App，要像一个懂你的健康助理。
5. **可读优先**：中老年人是核心用户之一，字体默认使用 **Medium（500）及以上字重**、足够的字间距与高对比度颜色（正文对比度 ≥ 7:1），**拒绝细体**；辅助文字不低于 13px，正文不低于 15px，数字型关键信息（时间、剂量）加粗加大。
6. **长者友好（减法设计）**：老人模式不是"另一个产品"，而是**"清爽版的标准产品"**——产品结构、底部导航、核心模块位置**保持一致**，只是隐藏掉复杂 / 商业 / 社交 / 推销类内容（如 AI 营销卡、健康商城、会员活动、科普、数据图表、家人动态等），让老人依然能无障碍地找到每一个核心功能，而无需重新学习。

## 五、核心用户画像与场景

### 5.1 用户画像

#### 画像一：自我管理型——"李先生，42 岁，高血压"
- 互联网从业者，生活节奏快，常出差；
- 3 年高血压病史，每日服用缬沙坦；
- 痛点：忘吃药、出差漏带药、续方临时跑医院；
- 期待：自动提醒 + 一键补货 + 出差前提醒带药。

#### 画像二：被关爱型——"王阿姨，68 岁，糖尿病+高血压"
- 退休，与子女分居；
- 合并用药 5 种，服用时间复杂；
- 痛点：吃错药、吃重药、子女不放心；
- 期待：清晰的每日用药卡 + 语音提醒 + 子女端同步。

#### 画像三：关怀者——"小张，30 岁，女儿"
- 在异地工作，父母独居；
- 痛点：不知道爸妈按时吃没吃药，不知道药快没了；
- 期待：家庭成员管理 + 服药状态远程可见 + 异常告警。

### 5.2 典型场景（故事化）

#### 场景一："李先生出差前"
> 周二晚上 21:00，李先生收到管家推送：**"明天上午您要出差 3 天，但降压药仅剩 2 天量，点这里立即补药，骑手 30 分钟送达。"** 一键下单，次日 8:00 前药品到家。

#### 场景二："王阿姨早晨吃药"
> 早 7:30，管家语音播报：**"王阿姨早上好，今天早饭前要吃的是：二甲双胍 1 片（白色圆形小药片）、缬沙坦 1 片（黄色胶囊）。吃完后请在手机上点一下'我吃了'。"** 未按时操作，8:30 再次提醒，9:00 自动通知女儿。

#### 场景三："小张查看父母情况"
> 小张打开管家"家人"Tab，看到**爸爸本周服药依从性 92%、妈妈 85%**。妈妈昨晚漏服一次，管家已自动发送温馨语音提醒。妈妈药品库存 ≤ 3 天自动触发一键续方。

## 六、产品核心设计

### 6.1 产品框架（四大核心模块）

```
┌─────────────────────────────────────────────────────────┐
│               慢病用药小管家                              │
├─────────────┬─────────────┬─────────────┬───────────────┤
│   药箱      │   日程      │   复诊      │   家人         │
│ Medication  │  Schedule   │  Follow-up  │   Family      │
│             │             │             │               │
│ · 药品档案  │ · 今日用药  │ · 续方提醒  │ · 成员管理    │
│ · 库存管理  │ · 打卡签到  │ · 在线问诊  │ · 远程查看    │
│ · OCR 录入  │ · 漏服补救  │ · 电子处方  │ · 关怀告警    │
│ · 相互作用  │ · 用药日历  │ · 复诊预约  │ · 代付代买    │
└─────────────┴─────────────┴─────────────┴───────────────┘
        ↑                                       ↑
        │     贯穿始终的 AI 管家（对话 + 主动服务）   │
        └───────────────────────────────────────┘
```

### 6.2 核心闭环

```
 ┌──── 问诊开方 ────┐         ┌──── 复诊续方 ────┐
 │                 │         │                 │
 ↓                 ↑         ↓                 ↑
购药下单 → 自动建立用药计划 → 每日提醒/打卡 → 库存预警 → 续方
                                       ↓
                                  依从性报告
```

**关键创新点**：用户在美团买药下单成功后，**系统基于订单自动创建用药计划**，无需用户重复录入，这是区别于所有竞品的关键体验。

## 九、核心用户流程图

### 9.1 首次使用流程

```
用户进入
   │
   ├─ 美团买药历史订单用户 ──→ 弹窗"发现您近期购买 XX 药，是否一键创建计划？" ──→ 确认 ──→ AI 生成计划 ──→ 设置提醒 ──→ 完成
   │
   └─ 新用户 ─────────────────→ 引导页 ──→ 拍药盒/手动添加 ──→ AI 解析 ──→ 确认用法 ──→ 设置提醒 ──→ 完成
```

### 9.2 每日用药流程

```
定时提醒（App/微信/短信/语音）
       │
       ↓
用户点开"今日用药卡"
       │
       ├─ 点"我吃了"  ──→ 库存 -1，记录打卡，AI 说"真棒，明天见"
       ├─ 点"稍后提醒" ──→ 30 分钟后再次提醒
       └─ 点"跳过"    ──→ 选择原因（忘了/不适/其他） ──→ AI 判断是否需要补救建议
```

### 9.3 续方闭环流程

```
触发条件（满足其一）：
  - 药品库存 ≤ 3 天
  - 电子处方剩余有效期 ≤ 7 天
  - 用户主动点"续方"
       │
       ↓
AI 预问诊（3 道题：近期症状/血压血糖/有无不适）
       │
       ↓
判断：原方续还是需要调整
       │
       ├─ 原方续 ──→ 跳转美团买药 ──→ 30 分钟送达
       └─ 需调整 ──→ 跳转在线问诊 ──→ 医生开方 ──→ 下单配送
```

### 10.3 老人模式（设计专题）

#### 10.3.1 设计哲学：做减法 ≠ 做极简

老人模式常见的两种错误做法：

- ❌ **只把字放大**：内容没变、视觉反而挤成一团，还是抓不到重点；
- ❌ **砍到只剩一件事**：界面骨架被打碎，老人切换回标准模式就不认识了，家人也没法教。

本产品的老人模式核心理念是：

> **"让老人看到的是清爽版的标准产品，而不是陌生的另一个产品"**

- **产品结构保持一致**：底部 5 个 Tab、核心模块的位置、命名、顺序都与标准模式完全一致；
- **去掉的是"干扰"**：AI 营销大卡、健康商城、会员权益、科普卡、家人动态、复杂图表、医生商业位；
- **加强的是"核心"**：今日用药更突出、打卡按钮更大、字体更粗、对比度更高、语音主导补全。

家人一次性配好老人模式、老人再学一次就能用，切回标准模式（比如子女代看）也完全熟悉。

#### 10.3.2 保留 vs 隐藏（黄金清单）

| 模块 | 标准模式 | 老人模式 |
| :--- | :---: | :---: |
| 今日用药卡（核心） | ✅ | ✅ |
| 一键打卡 / 跳过 | ✅ | ✅ |
| 用药日程 / 日历 | ✅ | ✅ |
| 药箱管理 | ✅ | ✅（简化详情） |
| 一键续方（被动入口） | ✅ | ✅ |
| 在线问诊 | ✅ | ✅ |
| 电子处方 | ✅ | ✅ |
| 家庭成员查看 | ✅ | ✅ |
| 基本健康数据（最近一次） | ✅ | ✅ |
| **AI 主动营销大卡**（"立即续方 ¥38" 类推送） | ✅ | ❌ |
| **健康商城推荐** | ✅ | ❌ |
| **慢病关爱会员活动** | ✅ | ❌ |
| **科普知识卡片** | ✅ | ❌ |
| **家人依从性榜单 / 动态** | ✅ | ❌ |
| **医生商业化推荐位** | ✅ | ❌ |
| **复杂数据图表 / 趋势** | ✅ | ❌（改为一句话结论） |
| **AI 管家悬浮球** | ✅ | ❌（改为 Tab 内入口，避免误触） |

**核心思路**：老人是**被动使用者**，不承担复购、营销、社交任务；涉及购买的动作由"家人代下单"承接。

#### 10.3.3 视觉规范（适度增强，避免布局扭曲）

| 元素 | 标准模式 | 老人模式 | 放大比例 |
| :--- | :--- | :--- | :---: |
| 正文字号 | 14px | **17px** | +21% |
| 关键信息字号（药名 / 时间） | 16px Medium | **20px Bold** | +25% |
| 字体字重 | Medium (500) | **Semibold (600)** | — |
| 行高 | 1.4 | **1.55** | — |
| 正文颜色 | `#374151` | **`#111827`** | — |
| 对比度 | ≥ 4.5:1 | **≥ 7:1（WCAG AAA）** | — |
| 按钮最小高度 | 40px | **52px** | +30% |
| 按钮文字（关键动作） | "我吃了" | **"✅ 我已经吃了"** | 语义化 |
| 触控目标 | ≥ 44×44px | **≥ 56×56px** | +27% |
| Tab 图标 | 22px | **26px** | +18% |

⚠️ **适度原则**：字号放大 **15%~25%** 即可，**不超过 30%**。过度放大会让卡片撑破、文字换行、按钮溢出、表格错位，反而降低可读性。

#### 10.3.4 首页模块对比

**标准模式首页（8 个模块）：**

1. 顶部问候 + 家人切换
2. **AI 管家主动营销卡**（续方提醒 + 价格 + 立即购买）

**老人模式首页（精简为 5 个模块）：**

1. 顶部问候 + 家人切换 ✅
2. ~~AI 管家主动营销卡~~ ❌ → 改为**一行简洁文字提示**："您的降压药还剩 6 天，告诉子女帮您续方"
3. 今日用药进度圈 ✅
4. 快捷服务（4 个 → **2 个**：问诊 / 家人；**续方需通过"服务"Tab 进入**）
5. **今日用药卡（核心）** ✅ ⬆️ 视觉位置提前

8. ~~科普小卡片~~ ❌

**整体感观**：结构一致、模块数 8 → 5、字更粗、按钮更大 —— 老人能一眼认出"就是同一个 App 清爽了一下"。

#### 10.3.5 其他页面的减法处理

| 页面 | 老人模式下隐藏的非核心内容 |
| :--- | :--- |

#### 10.3.6 语音主导原则

- **进入首页即主动语音播报**："早上好，王阿姨。现在是 8 点，该吃缬沙坦了。"
- **长按任意用药卡**（500ms）即可由管家朗读该药详情；

- **重要操作二次确认**：跳过 / 删除等操作由语音复述一次再执行。

## 十一、AI 能力应用设计

AI 是本产品的核心差异化，我们在 **4 个层次**应用 AI：

### 11.1 L1 感知层：OCR + 多模态识别

- **药盒识别**：OCR 提取药品通用名、规格；结合图像识别判断剂型；
- **处方识别**：识别医生手写处方（结合国家药品库做校对）；
- **身份证识别**：老人首次使用的便捷注册。

### 11.2 L2 理解层：医嘱结构化解析

- 医生输入"每日三次，每次 1 片，饭后半小时"——LLM 自动解析为结构化时间表：`{times: ["08:30","12:30","18:30"], relativeToMeal: "after", offsetMin: 30}`；
- 关键技术：**基于医药垂直领域微调的小参数 LLM（如 7B 级）+ 药品知识库 RAG**。

### 11.3 L3 决策层：智能推理与建议

- **漏服补救**：基于药品半衰期、是否为窄治疗指数药物，给出"立即补服"或"跳过这次"的建议；
- **药物相互作用**：基于药品知识图谱，实时计算冲突；
- **续方时机**：综合库存、处方有效期、用户出行日历（可选授权）智能提醒。

### 11.4 L4 交互层：对话式管家

- **人格化设计**：温和、关怀、专业，面向中老年友好；
- **主动对话**：早安、晚安、节日、复诊前后；
- **多模态输出**：文字 + 语音 + 图文卡片。

### 11.5 数据与安全底线

- 医疗建议类输出**均带免责声明**，严重问题引导就医；
- AI 不替代医生处方，续方必须经执业医师审核；
- 用户健康数据加密存储，遵循 **《个人信息保护法》《医疗健康数据安全规范》**。
