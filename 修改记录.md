## 2026-04-25 · 美团视觉对齐 + 老人模式 Switch 变形修复

### 背景

参考美团 App "我的"页截图，发现现版本视觉与美团主品牌存在两处不一致：

1. 字体栈仅 `PingFang SC + Microsoft YaHei`，在 HarmonyOS / Web 平台回落到细体无衬线，与美团 App 的厚实质感不一致。
2. 全局背景色 `#f4f5f7`（cool gray）属冷灰，美团 App 实测使用暖米白系背景，整体氛围更友好。

同时，用户反馈：**「点击老人模式后，老人模式的图标会变形」**。

排查发现：`index.css` 中 `.elder-mode button { min-height: 44px }` 这条**全局规则**把所有 `<button>` 元素都撑高到 44px。`Profile` 页的 `Switch` 组件原本是 `w-11 h-6`（44×24）的横向圆角开关，被这条规则强行拉到 44×44 正方形，thumb 圆点位置错位 → 看上去就是「图标变形」。这条规则原本意图是放大主操作按钮，但波及面过广，包括底部 Tab、Hero 头像、状态切换标签等也都被异常拉伸。

### 修改内容

| 维度 | 修改前 | 修改后 | 文件 |
|---|---|---|---|
| 字体栈 | `'PingFang SC', 'Microsoft YaHei', 'Hiragino Sans GB', system-ui` | `'PingFang SC', 'HarmonyOS Sans SC', 'HarmonyOS Sans', 'Hiragino Sans GB', 'Microsoft YaHei', 'WenQuanYi Micro Hei', sans-serif` + 数字优先 `'DIN Alternate'` | `src/index.css` |
| 全局背景 | `bg-gray-50` (#f9fafb) | 美团暖白 `#fff7ec`（与美团 App 暖底一致） | `src/components/Layout.tsx` + `src/index.css` (body) |
| 滚动条 thumb | `#d1d5db` | `#f0d9b8`（暖米色） | `src/index.css` |
| **老人模式 Switch 变形** | `.elder-mode button { min-height: 44px }` 全局规则 | **删除该规则**；只保留 `.elder-mode button.bg-brand-600`、`.elder-mode .btn-primary` 等主操作按钮的 `min-height: 52px` | `src/index.css` |

### 视觉对齐细则

- 美团品牌色仍以橙色 `#ff8000` 为主操作按钮 / 渐变 Hero，符合既有 brand 系统；
- 暖米白 `#fff7ec` 选自美团 App "钱包/会员中心"块的卡片间隙底色，柔和不刺眼；
- 字体增加 `HarmonyOS Sans SC` 让安卓鸿蒙端有原生质感；数字优先 `DIN Alternate` 让"今日用药 2/4""血压 128"等数字呈现圆润、稳定的美团调性。

### 影响范围

- 【视觉】所有页面背景从冷灰 → 暖米白；首屏视觉氛围更接近美团；
- 【可读性】字体在 HarmonyOS / 老旧 Windows 下不会回退到默认细体；
- 【Bug 修复】老人模式下 `Switch` 恢复 44×24 横向开关；其他被误伤的 `<button>`（Hero 头像 40×40、Tab 按钮等）尺寸恢复正常；
- 【主按钮无影响】"我吃了""一键续方"等主按钮仍按 52px 放大执行，体验不退化。

### 验收点

- [x] `npm run build` 通过，0 error 0 warning；
- [x] 标准模式：首页 Hero、卡片间留白与美团 App 接近；
- [x] 老人模式："我的 → 老人模式"开关恢复横向滑动条，thumb 位置正确；
- [x] 老人模式：底部 Tab 不再被异常拉高，Hero 头像保持圆形 40×40。

## 2026-04-24 · 老人模式回归"清爽版"（撤销过度极简）

### 背景

第一版老人模式做"减法"过头，问题：
1. 独立 3 Tab 布局 → 与标准模式产品结构断裂；
2. 首页"一屏一事"极简化 → 本质上变成"另一个产品"；
3. 字号放大 50%~100% → 布局多处溢出 / 留白巨大；
4. 按钮 72px → 视觉过重，与美团 App 一致性差。

用户反馈：**"老人模式需要做减法，但现在减得太多，界面被破坏了。老人模式应该在原产品基础上减少一些复杂功能，比如 AI 买药提示。"**

### 修改内容

| 维度 | 修改前 | 修改后 |
|---|---|---|
| 路由结构 | App.tsx 按 `elderMode` 分流到独立 `ElderLayout` | 统一 `Layout`，由各页面内部按 `elderMode` 条件渲染 |
| 底部导航 | 老人模式 3 Tab（用药/家人/我的） | 5 Tab 不变，仅图标 22→28、字号加大、字重加粗 |
| 字号放大 | +50%~100%（text-3xl → 46px） | +15%~25%（text-3xl → 34px），不超过 30% |
| 主按钮高度 | 72px | 52px |
| 全局字重 | 强制 `font-weight: 600 !important` | 不强制；仅 `<b>/<strong>/h1~h3` 加粗 |
| Home 页 | 独立 `ElderHome.tsx`「一屏一事」 | 复用 `Home.tsx`，老人模式下隐藏 AI 营销卡 / 家人动态 / 科普 / 空腹血糖卡 |
| 快捷服务 | 老人模式只显示 1 个 | 4 → 2（保留续方 + 问诊） |
| 续方提示 | 橙色营销卡（"立即续方 ¥38"） | 沉稳大按钮"药快用完了，点此续方" |
| 悬浮 AI 球 | 老人模式仍显示 | 老人模式隐藏（防误触） |
| 删除文件 | — | `src/components/ElderLayout.tsx`、`src/pages/ElderHome.tsx` |

### "保留 / 隐藏" 黄金清单

| 页面 | 老人模式隐藏 | 老人模式保留 |
|---|---|---|
| Home | AI 管家营销卡、家人用药状态、科普卡、空腹血糖卡 | Hero / 进度环 / 今日用药 / 血压卡 |
| Medicine | 搜索框、AI 药物相互作用入口、已生效处方统计 | 添加按钮、药品数 / 即将用完、药品列表 |
| Schedule | 月视图打卡奖励卡、柱状图、健康数据趋势、导出 PDF | 3 Tab 结构、今日列表、月日历、AI 周报建议（改为一句话） |
| Services | 慢病专区商城、慢病关爱会员卡 | 续方、问诊、电子处方、复诊预约 |
| Family | 家人关怀 Hero 大卡、AI 管家洞察 | 成员列表、切换、依从性、远程提醒 |
| Profile | 老人模式开关下的 3 个解释小格 | 一句描述文案 + Switch 开关 |

### 影响范围

- 【产品结构】两种模式共用同一套路由与布局，结构一致性回归；
- 【可学习性】子女教爸妈使用时不再有"两个产品"的认知断层；
- 【视觉】适度放大保持美团既有视觉系统，不破坏圆角 / 阴影 / 间距比例。

### 验收点

- [x] `tsc -b && vite build` 通过；
- [x] 老人模式仍是 5 Tab，所有页面入口可达；
- [x] 老人模式下营销 / 复杂图表 / 商城类内容已隐藏；
- [x] 标准模式恢复原貌，所有功能完整。

## 2026-04-23 · 老人模式首次迭代（已被覆盖）

### 背景

用户首次反馈："老人模式不应该只放大字体，更应该减少一些复杂内容；同时字体有点纤细不够清晰。"

### 当时的实现（已被 04-24 版本覆盖）

- 创建 `ElderLayout.tsx`：独立 3 Tab 布局；
- 创建 `ElderHome.tsx`：「一屏一事」极简首页；
- `index.css` 强制全局字重 600；
- 老人模式字号放大 50%~100%；
- 主按钮 72px。

### 教训

- 「做减法 ≠ 做极简」：过度简化会让产品失去原有的导航和服务能力，反而**增加**学习成本；
- 老人模式应是**清爽版的标准产品**：结构、模块位置、底部导航全部一致，仅在视觉与内容密度上做减法；
- 视觉放大需要克制：参考 iOS 辅助功能 / WCAG AAA，放大 15~25% 已能显著改善阅读，超过 30% 反而破坏布局节奏。
