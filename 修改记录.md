## 2026-04-25 (深夜) · 全产品橙→美团黄统一收官（按钮/图标/悬浮球/进度环）

### 背景

用户对上一版的反馈：

> 「之前打的橙色，我只是希望他变成黄色，但是现在直接修改没有了，同时等些图标等颜色，我都希望修改成黄色，**所有橙色图标变成美团黄**。」

复盘上一版的失误：

- 上一版只把 **5 个 Hero 大区**（首页/服务/我的/电子处方/续方导入卡）从橙渐变改成了 `meituan-* 黄渐变`。
- 但**主按钮 `bg-brand-600`、所有图标 `text-brand-600`、AI 悬浮球、Chat AI 头像、进度环**仍然是橙色 `#ff8000`。
- 形成了「**Hero 黄、按钮橙、图标橙**」的撕裂感，跟用户期望的"全黄"完全不一致。

本次目标：**品牌色彻底从橙变黄**，橙色仅保留在「警告语义」（库存预警、漏服、依从性<80%）。

### 设计决策

#### 1. 调色板从橙系切到黄系（一次性切换 brand）

| 色阶 | 上一版（橙） | 本版（美团黄） | 用途 |
| ------- | ----------- | -------------- | ---- |
| brand-50 | `#fff7ec` 浅橙白 | `#FFFCEA` 浅米黄 | 卡片浅底、AI 信息卡 |
| brand-100 | `#ffe8cc` | `#FFF6CC` | 浅黄装饰 |
| brand-200 | `#ffd199` | `#FFEB99` | 边框、装饰渐变 |
| brand-300 | `#ffb866` | `#FFDF66` | Hero 渐变开始色 |
| brand-400 | `#ff9e3d` | `#FFD333` | Hero 渐变中段 |
| brand-500 | `#ffc300` | `#FFD200` | 美团 logo 黄、Hero 终点 |
| brand-600 | `#ff8000` 橙 | **`#FFC500` 美团主黄** | 按钮主色、强调 |
| brand-700 | `#e66b00` | **`#A16207` 深金棕** | 文字/图标可读色（白底高对比） |

> **关键技巧**：黄色按钮在白底上「面积大对比够」（`bg-brand-600`），但黄色文字/图标在白底上「面积小对比差」。所以让 **600 = 鲜黄、700 = 深金棕**，按钮用 600，文字/图标用 700，两全其美。

并**删除独立的 `meituan` 调色板**——上一版引入它是为了 Hero 渐变，本次合并后不再需要双调色板，全部走 `brand`，调色板更收敛。

#### 2. 按钮配色对齐美团 App：黄底 + 深字 + 加粗

参考美团首页的「搜索」按钮：纯亮黄底 + 黑色字 + 加粗。

| 之前 | 现在 |
| ---- | ---- |
| `bg-brand-600 text-white font-medium` (橙底白字) | `bg-brand-600 text-gray-900 font-bold/font-semibold` (黄底深字加粗) |

涉及 **24 处主要按钮**（覆盖 11 个文件）：
- 首页：立即续方 / 已服 / 主操作 (`Home.tsx`)
- 药品列表：扫码加药 / Tab 选中 (`Medicine.tsx` / `Schedule.tsx`)
- 加药页：常用时间 chip / 一键导入 / 确认按钮 / 下一步 / Badge (`MedicineAdd.tsx`)
- 详情页：管理提醒按钮 (`MedicineDetail.tsx`)
- 续方流程：选项 chip / 下一步 / 提交订单 (`Refill.tsx`)
- 处方页：购药按钮 (`Prescription.tsx`)
- 服务页：标签 (`Services.tsx`)
- 家人页：邀请按钮 / 添加家人 (`Family.tsx`)
- 咨询页：复诊按钮 (`Consultation.tsx`)
- AI 聊天：发送按钮 / 用户消息气泡 / 卡片行动按钮 (`Chat.tsx`)

#### 3. 文字 / 图标提升对比度：`text-brand-600` → `text-brand-700`

`brand-600 = #FFC500` 鲜黄做文字色对比度太低（WCAG AA 不达标），全局批量替换到 `brand-700 = #A16207` 深金棕。涉及约 **24 处**：
- Tab 选中、Stat 数字、价格、icon 颜色、标签 chip、链接、副标题等。

#### 4. AI 悬浮球 / Chat AI 头像 / 进度环 → 黄色

- **AI 悬浮球** (`AIButlerFab.tsx`)：`from-brand-500 to-brand-600 text-white` → `from-brand-400 to-brand-500 text-gray-900`
  - 之前是橙色圆球，现在是亮黄渐变 + 深色 Sparkles 图标，与美团黄整体氛围一致。
- **Chat AI 头像 / 用户气泡前 AI 头像**（3 处）：`from-brand-500 to-orange-500 text-white` → `from-brand-400 to-brand-500 text-gray-900`
- **首页今日用药进度环**：`stroke="#ff8000"` → `stroke="#FFC500"`
- **家人/医生头像装饰渐变**：`from-brand-100 to-orange-100` → `from-brand-100 to-brand-200`

#### 5. 颜色语义清洗（重要）

**保留**了所有「警告语义」相关的橙色：

- `bg-orange-50 text-meds-warn`：库存不足提示（首页、药品列表、详情页）
- 月日历"部分服药"状态：`status==='partial' bg-orange-400`
- 家人依从性 < 80% 进度条：`from-orange-400 to-orange-500`
- 药物冲突卡片：`MedicineInteraction.tsx` 的 `border-orange-100 bg-orange-50/50`

**理由**：在医疗场景中，橙色 = 警告/留意 是国际通用色，与品牌色解耦才能保证清晰传达「这是异常、需要关注」的信号。如果连警告也变黄，用户对"哪里出问题了"的视觉敏感度会下降。

### 修改清单

- 调色板
  - `tailwind.config.js`：brand 50–900 全套从橙改为黄系；删除 meituan 调色板
- 主按钮（24 处）
  - 11 个文件，统一 `bg-brand-600 text-white` → `bg-brand-600 text-gray-900` + 加粗
- 图标 / 文字（24 处）
  - 12 个文件，统一 `text-brand-600` → `text-brand-700`
- Hero 渐变（7 处）
  - `from-meituan-200 via-meituan-300 to-meituan-500` → `from-brand-300 via-brand-400 to-brand-500`
- 装饰元素
  - AI 悬浮球：橙渐变 → 黄渐变 + 深色图标
  - Chat 3 处 AI 头像：橙渐变 → 黄渐变
  - Home 进度环 stroke：`#ff8000` → `#FFC500`
  - Family / Consultation 头像装饰：`to-orange-100` → `to-brand-200`

### 影响 / 验证

- 首页 Hero、悬浮球、立即续方按钮、用药进度环、Stat 数字、Tab 选中态全部统一为美团黄系，视觉氛围与美团 App 一致。
- 警告语义橙色（库存预警 / 漏服 / 依从性低）保留并独立，颜色含义不再混淆。
- `npm run build` 成功，0 error / 0 warning，CSS 体积 27 KB（gzip 5.89 KB）。
- 已通过 HMR 实时验证。

### 后续可选

- 主按钮统一加 `.btn-primary` 类，便于老人模式下的 52px 按钮规则更精准命中（当前依赖 `bg-brand-600` 检查）。
- 黄色 Switch 在浅色背景上对比度可再提升：考虑给 Switch thumb 加 1px 灰色描边。

## 2026-04-25 (晚) · Hero 换美团黄渐变 + Switch 圆球畸形彻底修复

### 背景

视觉走查 + 用户反馈：

1. **背景橙黄希望换成美团类的黄**：当前 Hero 是 `from-brand-500 via-brand-600 to-orange-500` 的纯橙红渐变，与美团 App 顶部"亮黄渐变"差距明显，整体氛围偏深、偏暖红。需要对齐美团黄系（如外卖首页、会员中心顶部黄）。
2. **老人模式按钮畸形（再次出现）**：开启老人模式后，"我的 → 老人模式"右侧的 Switch 变成一个**橙色大圆**而非横向滑块。

### 问题 2 根因复盘

上一版我把 CSS 全局规则 `.elder-mode button { min-height: 44px }` 删除后，又新增了：

```css
.elder-mode button.bg-brand-600 { min-height: 52px; }
```

`Switch` 在开启状态下的 `<button>` 上正好挂着 `bg-brand-600`（用作开启色），结果**再次被命中**：被强制拉成 `44×52` 圆角竖长方形 + `rounded-full` → 视觉上就是一个大橙圆。

**根本性修复**：抛弃"靠 `bg-brand-600` 隐式识别主操作按钮"的思路，改为**显式标识**：
- 主操作按钮要在老人模式被加大 → 加 `.btn-primary` 类（未来按需添加）
- Switch / Tab / 头像等小型按钮 → 不带 `.btn-primary` 即不会被加大
- Switch 额外加 `.toggle-switch` 类，CSS 用 `!important` 强制锁定 `44×24`，三层保险

### 修改内容

#### A. 调色板：`tailwind.config.js`

新增美团黄色调色板，与既有 `brand`（橙）共存：

```js
meituan: {
  50:  '#FFF9D6',
  100: '#FFF4B0',
  200: '#FFEB7A',
  300: '#FFE140',
  400: '#FFD61F',  // 美团 App Hero 主黄
  500: '#FFC600',
  600: '#FFB400',
  700: '#E69F00',
}
```

设计原则：「**黄底 + 橙按钮**」双色搭配 = 美团 App 经典配色。
- 大区背景（Hero / 营销渐变卡）走 `meituan` 黄
- 主操作按钮（"我吃了"/"立即续方"/"添加"等）保留 `brand-600` 橙

#### B. Hero 大区批量替换

| 文件 | 位置 | 替换前 | 替换后 |
|---|---|---|---|
| `pages/Home.tsx` | 顶部 Hero | `from-brand-500 via-brand-600 to-orange-500 text-white` + `bg-white/20` 头像 | `from-meituan-200 via-meituan-300 to-meituan-500 text-gray-900` + `bg-white/70` 头像 |
| `pages/Services.tsx` | 顶部 Hero | 同上 | 同上 |
| `pages/Profile.tsx` | 顶部 Hero | 同上 | 同上 |
| `pages/Schedule.tsx` | 月视图"连续打卡 18 天"卡 | `from-brand-500 to-orange-500 text-white` | `from-meituan-200 via-meituan-300 to-meituan-500 text-gray-900` |
| `pages/Family.tsx` | 邀请家人弹窗的邀请码卡 | 同上 | 同上 |
| `pages/Prescription.tsx` | 处方头部 | 同上 | 同上 |
| `pages/MedicineAdd.tsx` | "AI 智能添加"提示卡 | 同上 | 同上 |

#### C. 文字反转：白字 → 深灰字

黄色背景上**白字对比度差**，按 WCAG 应改深色字：
- 主标题：`text-white font-semibold` → `text-gray-900 font-bold`
- 次要文字：`text-white opacity-90` → `text-gray-700` / `text-gray-800`
- 头像背景：`bg-white/20`（黄底上几乎透明） → `bg-white/70 shadow-sm`（白底浮起，立体）
- 标签胶囊：`bg-white/20` → `bg-white/70 text-gray-800`

#### D. 保留橙色的位置（差异化）

| 元素 | 文件 | 理由 |
|---|---|---|
| 主操作按钮 `bg-brand-600` | 全局 | 黄底 + 橙按钮 = 美团双色调，强行动召唤 |
| Hero 进度环 `stroke="#ff8000"` | Home | 黄底 + 橙环对比强 |
| AI 管家悬浮球 | AIButlerFab | 圆形小图标，黄底白字辨识不清 |
| Chat AI 头像（圆形 32×32） | Chat | 同上 |

#### E. Switch 三层保险修复

1. **CSS 显式标识**：

```css
/* 仅 .btn-primary 才被认为是主操作按钮 */
.elder-mode .btn-primary {
  min-height: 52px;
  font-weight: 700;
  letter-spacing: 0.5px;
}

/* 切换开关豁免：保持 44×24 横向 toggle 形态（!important 锁死） */
.elder-mode .toggle-switch {
  min-height: 24px !important;
  height: 24px !important;
  width: 44px !important;
}
```

2. **组件加 className**：

```tsx
<button
  className={cx(
    'toggle-switch relative w-11 h-6 rounded-full transition shrink-0',
    on ? 'bg-brand-600' : 'bg-gray-200',
  )}
  aria-pressed={on}
>
```

3. **去除 `bg-brand-600` 命中**：CSS 选择器不再使用 `button.bg-brand-600`，从根上避免任何"开启状态橙色按钮"被误判为主操作按钮。

### 验收点

- [x] `npm run build` 通过；
- [x] Home / Services / Profile 顶部呈现美团亮黄渐变，文字深色清晰可读；
- [x] 头像/标签在黄底上以白色半透明浮起，立体感正确；
- [x] 老人模式开关恢复 44×24 横向滑块，开启状态橙色背景 + 白色 thumb 圆点正常；
- [x] 老人模式下，底部 Tab、Hero 头像、所有非主按钮保持原比例。

### 影响范围

- 视觉氛围全面对齐美团 App，**美团黄 + 美团橙** 双色调系统建立；
- Switch、Tab、小图标按钮在老人模式下**永不变形**（三层保险）；
- 未来若希望某个新按钮在老人模式被加大，只需加 `.btn-primary` 类，规则清晰可控。

## 2026-04-25 · 美团视觉对齐 + 老人模式 Switch 变形修复

### 背景

参考美团 App "我的"页截图，发现现版本视觉与美团主品牌存在两处不一致：

1. 字体栈仅 `PingFang SC + Microsoft YaHei`，在 HarmonyOS / Web 平台回落到细体无衬线，与美团 App 的厚实质感不一致。
2. 全局背景色 `#f4f5f7`（cool gray）属冷灰，美团 App 实测使用暖米白系背景，整体氛围更友好。

同时，用户反馈：**「点击老人模式后，老人模式的图标会变形」**。

排查发现：`index.css` 中 `.elder-mode button { min-height: 44px }` 这条**全局规则**把所有 `<button>` 元素都撑高到 44px。`Profile` 页的 `Switch` 组件原本是 `w-11 h-6`（44×24）的横向圆角开关，被这条规则强行拉到 44×44 正方形，thumb 圆点位置错位 → 看上去就是「图标变形」。这条规则原本意图是放大主操作按钮，但波及面过广，包括底部 Tab、Hero 头像、状态切换标签等也都被异常拉伸。

### 修改内容

| 维度 | 修改前 | 修改后 | 文件 |
|---|---|---|---|
| 字体栈 | `'PingFang SC', 'Microsoft YaHei', 'Hiragino Sans GB', system-ui` | `'PingFang SC', 'HarmonyOS Sans SC', 'HarmonyOS Sans', 'Hiragino Sans GB', 'Microsoft YaHei', 'WenQuanYi Micro Hei', sans-serif` + 数字优先 `'DIN Alternate'` | `src/index.css` |
| 全局背景 | `bg-gray-50` (#f9fafb) | 美团暖白 `#fff7ec`（与美团 App 暖底一致） | `src/components/Layout.tsx` + `src/index.css` (body) |
| 滚动条 thumb | `#d1d5db` | `#f0d9b8`（暖米色） | `src/index.css` |
| **老人模式 Switch 变形** | `.elder-mode button { min-height: 44px }` 全局规则 | **删除该规则**；只保留 `.elder-mode button.bg-brand-600`、`.elder-mode .btn-primary` 等主操作按钮的 `min-height: 52px` | `src/index.css` |

### 视觉对齐细则

- 美团品牌色仍以橙色 `#ff8000` 为主操作按钮 / 渐变 Hero，符合既有 brand 系统；
- 暖米白 `#fff7ec` 选自美团 App "钱包/会员中心"块的卡片间隙底色，柔和不刺眼；
- 字体增加 `HarmonyOS Sans SC` 让安卓鸿蒙端有原生质感；数字优先 `DIN Alternate` 让"今日用药 2/4""血压 128"等数字呈现圆润、稳定的美团调性。

### 影响范围

- 【视觉】所有页面背景从冷灰 → 暖米白；首屏视觉氛围更接近美团；
- 【可读性】字体在 HarmonyOS / 老旧 Windows 下不会回退到默认细体；
- 【Bug 修复】老人模式下 `Switch` 恢复 44×24 横向开关；其他被误伤的 `<button>`（Hero 头像 40×40、Tab 按钮等）尺寸恢复正常；
- 【主按钮无影响】"我吃了""一键续方"等主按钮仍按 52px 放大执行，体验不退化。

### 验收点

- [x] `npm run build` 通过，0 error 0 warning；
- [x] 标准模式：首页 Hero、卡片间留白与美团 App 接近；
- [x] 老人模式："我的 → 老人模式"开关恢复横向滑动条，thumb 位置正确；
- [x] 老人模式：底部 Tab 不再被异常拉高，Hero 头像保持圆形 40×40。

## 2026-04-24 · 老人模式回归"清爽版"（撤销过度极简）

### 背景

第一版老人模式做"减法"过头，问题：
1. 独立 3 Tab 布局 → 与标准模式产品结构断裂；
2. 首页"一屏一事"极简化 → 本质上变成"另一个产品"；
3. 字号放大 50%~100% → 布局多处溢出 / 留白巨大；
4. 按钮 72px → 视觉过重，与美团 App 一致性差。

用户反馈：**"老人模式需要做减法，但现在减得太多，界面被破坏了。老人模式应该在原产品基础上减少一些复杂功能，比如 AI 买药提示。"**

### 修改内容

| 维度 | 修改前 | 修改后 |
|---|---|---|
| 路由结构 | App.tsx 按 `elderMode` 分流到独立 `ElderLayout` | 统一 `Layout`，由各页面内部按 `elderMode` 条件渲染 |
| 底部导航 | 老人模式 3 Tab（用药/家人/我的） | 5 Tab 不变，仅图标 22→28、字号加大、字重加粗 |
| 字号放大 | +50%~100%（text-3xl → 46px） | +15%~25%（text-3xl → 34px），不超过 30% |

## 2026-04-23 · 老人模式首次迭代（已被覆盖）

### 背景

用户首次反馈："老人模式不应该只放大字体，更应该减少一些复杂内容；同时字体有点纤细不够清晰。"

### 当时的实现（已被 04-24 版本覆盖）

- 创建 `ElderLayout.tsx`：独立 3 Tab 布局；
- 创建 `ElderHome.tsx`：「一屏一事」极简首页；
- `index.css` 强制全局字重 600；
- 老人模式字号放大 50%~100%；
- 主按钮 72px。

### 教训

- 「做减法 ≠ 做极简」：过度简化会让产品失去原有的导航和服务能力，反而**增加**学习成本；
- 老人模式应是**清爽版的标准产品**：结构、模块位置、底部导航全部一致，仅在视觉与内容密度上做减法；
- 视觉放大需要克制：参考 iOS 辅助功能 / WCAG AAA，放大 15~25% 已能显著改善阅读，超过 30% 反而破坏布局节奏。
